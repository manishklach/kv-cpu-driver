I. Physical Die Micro-Floorplan

QUADRANT A

NMCE Compute Engine

  • Unit: D-wide DPU (64/128/256 MACs)[cite: 1]
  • Buffer: 16KB Query Vector SRAM (QVB)[cite: 1]
  • Logic: Piecewise Linear exp() (8 segments)[cite: 1]
  • Traffic Reduction: 25.6x - 128x vs Baseline[cite: 1]
QUADRANT B

HEPC Policy FSM

  • Registers: MMIO Step-Advance (50ns latency)[cite: 1]
  • Logic: 64k-entry Pipelined Priority Scorer[cite: 1]
  • DMA: Independent Evict/Prefetch Channels[cite: 1]
  • Cycles: Scan -> Evict -> Prefetch[cite: 1]
QUADRANT C

RTBD Tag Store

  • Storage: 240-bit SRAM CAM entries[cite: 1]
  • Multi-Tenancy: Request-ID Tagging (16b)[cite: 1]
  • Isolation: Hardware Reference Counting[cite: 1]
  • Lookup: Single-cycle address resolution[cite: 1]

III. Silicon Logic: Priority Scoring Function

The HEPC recomputes block importance every decode step using fixed-function arithmetic logic[cite: 1]:

// Hardware Implementation (Single Cycle per Block)
priority_score[B_i] = (w_r * Recency_Counter) +
                     (w_f * Frequency_Counter) +
                     (w_s * max(0, W - (curr_step - last_access))) +
                     (w_d * is_prefix_flag);

IV. OS-Level Control Plane (Pillar IV)

The kernel module exposes semantic memory hints to userspace runtimes (vLLM/SGLang)[cite: 1]:

  • MADV_KV_HOT: Set priority score to 0xFFFF (Exclude from eviction)[cite: 1]
  • MADV_KV_EVICT: Immediate background DMA transfer to T2/T3[cite: 1]
  • MADV_KV_PREFETCH: Immediate stage-to-T1 for predicted tokens[cite: 1]
  • IORING_OP_KV_STAGE: Async zero-copy migration with completion queue[cite: 1]
