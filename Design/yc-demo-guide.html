Inference is becoming memory-orchestration bound. We've built the semantic control plane for the LLM era.

01. The Setup

Initializing the Control Plane

We load the driver in Mock Mode to demonstrate the architectural bridge without requiring physical hardware.

$sudo insmod kv_cpu.ko mock=1
$ls -l /dev/kvcpu0
Talking Point: "Notice /dev/kvcpu0. This isn't just a buffer; it's the industry's first standardized 'Inference Control Plane.' It allows the OS to understand what is happening inside the LLM runtime."

02. The Signaling

Decode Step Synchronization

During inference, the runtime signals every decode iteration.

$./tools/kvctl step 256
Talking Point: "As the model generates tokens, we signal the 'Step' to the kernel. In a real KV-CPU, this triggers an autonomous hardware scan that re-scores the priority of every block in memory in nanoseconds."
Semantic Policies (HOT/EVICT)

Protecting shared prefixes from "blind" OS eviction.

$./tools/kvctl hot 0x7f001000 4096
Talking Point: "Standard Linux is 'semantic-blind'—it might evict a critical shared prefix. We signal it as HOT, ensuring the hardware locks it into the fastest memory tier."

03. The Moat

Proactive Orchestration

Eliminating prefetch latency by signaling future needs.

$./tools/kvctl prefetch 0x8f0000 4096 512
Talking Point: "Finally, we signal the future. By hinting exactly when a block will be needed, the KV-CPU migrates data in the background, completely hiding the latency of memory tiering."
"The KV-CPU team is attacking the context-window bottleneck where it is most expensive: the memory orchestration layer. By moving eviction logic from slow software to step-aware hardware, they are building the 'Control Plane for AI Infrastructure'—a high-leverage position in the stack."
